Hashes vs Objects

11/23/2014

This week we started to learn some basic JavaScript. JavaScript has some similarities and differences from the previous programming language we’ve in Phase 0 Ruby. One thing that really stood out to me when first learning the JavaScript this past week was the similarity between JavaScript’s objects and Ruby Hashes. I want to take a little time comparing these two things.

If you been following along with me during my time in Phase 0 you would know that Rudy’s Hash is a collection object with a key value pairs. Similar to hashes objects in JavaScript use key value pairs but instead of keys they use properties.

Just like hashes in Ruby value in the pairings in JavaScript can be everything even function. However, the properties in JavaScript not as flexible they can’t be any object like in Ruby. Properties in JavaScript are strictly used a reference to the values.

One last thing since Hashes are objects in Ruby they have methods that are built in to the class while allows for various manipulation to be done with different method calls. Objects in JavaScript are not as robust as the Hash class in Ruby so it might be more difficult to do some the same manipulation.

Just a quick disclaimer this week I been working with JavaScript for less than a week so I’m not sure if all my facts are correct.
